# Contributing to VORTEX

We welcome contributions that improve the VORTEX project!

## Getting Started
- Fork the repo, create a feature branch from `main`
- Make clear, modular commits that are easy to review
- Write tests where applicable and update docs as needed

## Code Style
- Use clear, descriptive naming and favor maintainable patterns
- Keep commits focused and well-documented
- PRs must pass CI and review before merging

## Communication
- Open issues for bugs or feature requests with clear context
- Propose architectural changes before implementing large features

Thank you for contributing!
